# Contributing to the SCP Smart Contracts Documentation

Thank you for helping improve the documentation! This repository is dedicated exclusively to documents — no source code belongs here.

## What belongs in this repository

- Markdown (`.md`) documents
- Diagrams in Markdown-embeddable formats (Mermaid, SVG, PNG)
- Reference tables and API descriptions

## What does **not** belong here

- Smart contract source code (`.sol`)
- Compiled artifacts or ABIs
- Scripts or configuration files unrelated to documentation

## How to contribute

1. **Fork** the repository and create a new branch from `main`.
2. Add or edit documents in the relevant subdirectory under `docs/`:
   - `docs/architecture/` — design and architecture documents
   - `docs/contracts/` — smart contract specifications
   - `docs/guides/` — user and developer guides
   - `docs/api/` — API references
3. Use **Markdown** for all documents.
4. Write in clear, concise language. Prefer short paragraphs and bullet lists.
5. Link to related documents where helpful.
6. Open a **pull request** against `main` with a clear description of your changes.

## Style guidelines

- Use title case for headings (e.g., `## Getting Started`, not `## getting started`).
- Use backticks for inline code and contract names (e.g., `MyContract`).
- Keep lines shorter than 120 characters where possible.
- Spell-check before submitting.

## Review process

All pull requests require at least one reviewer approval before merging. Reviewers will check for accuracy, clarity, and adherence to the guidelines above.
